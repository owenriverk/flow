# Design: lateboof.com — flow gauge landing page

Generated by /brainstorming on 2026-06-28
Status: APPROVED

## Problem Statement

The Flow bot (flow@lateboof.com) has no public face. Paddlers who hear about it have nowhere to learn how it works or find the list of supported gauges. This site solves both with two pages.

## Goals

- Explain the service in one glance (what it is, how to use it)
- Show live flows for all ~70 curated gauges with AW/BCW color coding
- Feel like a homebuilt tool, not a SaaS — simple type, no gradients, no hero images
- No logins, no accounts, nothing to install

## Architecture

```
lateboof.com (Cloudflare Pages — static HTML/CSS/vanilla JS)
    │
    └── fetches on load + every 10 min ──▶ Supabase REST API (anon key)
                                                │
                                        gauges table
                                                │
                                    pg_cron every 15 min
                                                │
                                  refresh-gauges Edge Function
                                                │
                              ┌─────────────────┼──────────────────┐
                           USGS IV API       WSC API           CDEC API
```

No build step. No framework. No bundler. Two `.html` files, one `.css`, one `.js`.

## Data Layer

### Supabase table: `gauges`

| column          | type        | notes                                      |
|-----------------|-------------|---------------------------------------------|
| `key`           | text PK     | canonical alias key, e.g. `middle fork salmon` |
| `name`          | text        | display name, e.g. `MF Salmon`             |
| `location`      | text        | e.g. `MF Lodge, ID`                        |
| `source`        | text        | `usgs` / `wsc` / `cdec`                   |
| `site`          | text        | gauge ID                                   |
| `discharge`     | numeric     | current flow (null if unavailable)         |
| `discharge_unit`| text        | `cfs` or `cms`                             |
| `stage`         | numeric     | stage height (null if unavailable)         |
| `stage_unit`    | text        | `ft` or `m`                               |
| `reading_time`  | timestamptz | timestamp from the upstream gauge          |
| `updated_at`    | timestamptz | when the Edge Function last polled         |
| `low`           | numeric     | flow below this = red (null = no range)    |
| `high`          | numeric     | flow above this = blue (null = no range)   |
| `gauge_url`     | text        | link to USGS/WSC/CDEC source page          |
| `text_key`      | text        | shortest alias to text (e.g. `chilko`)     |

Rows are deduped: aliases that point to the same gauge (e.g. `mf salmon` / `middle fork salmon`) share one row under the canonical key. The `text_key` column holds the shortest usable alias for display in the directory.

### Seeding

- **Flow ranges (US rivers):** looked up via AW's public reach JSON endpoint per river; stored in `low` / `high` in native units (cfs for US).
- **Flow ranges (Canadian rivers):** sourced manually from BC Whitewater / RiverApp; stored in `low` / `high` in cms.
- **Rivers with no range data:** `low` and `high` are null; row shows flow but no color.

### Refresh

A Supabase Edge Function (`refresh-gauges`) imports the existing `usgs.ts` / `wsc.ts` / `cdec.ts` fetchers, iterates the canonical gauge list, and upserts `discharge`, `stage`, `reading_time`, `updated_at`. `pg_cron` calls it every 15 minutes. The frontend re-fetches every 10 minutes via `setInterval`.

The Edge Function reads the canonical gauge list from a committed `gauges-seed.json` (same shape as `aliases.json` but deduped and augmented with `low`/`high`/`gauge_url`/`text_key`).

## Page 1 — `index.html` (explainer)

Single column, clean type on off-white (`#f9f7f4`). No images. No nav beyond a link to the gauge directory.

```
lateboof.com

Flow
────────────────────────────────────────
Text a river name from your InReach.
Get the current flow back.

HOW IT WORKS

1. Add flow@lateboof.com as an InReach contact
2. Text a gauge name — "middle fork salmon",
   "grand canyon", "chilko"
3. Get the flow back in seconds, over satellite

No app. No signal. No login.

[See all gauges →]
────────────────────────────────────────
Built for paddlers, by a paddler.
```

Typography: system sans-serif stack. Large, readable. The "HOW IT WORKS" block is the only real content — keep it that simple.

## Page 2 — `gauges.html` (directory)

A plain sortable table. One row per canonical gauge, sorted alphabetically by river name. A small legend above the table explains the color coding.

### Columns

| River | Location | Flow | Text this | Updated |
|---|---|---|---|---|
| MF Salmon | MF Lodge, ID | 2,100 cfs | `mf salmon` | 8 min ago |

- **River** links to `gauge_url` (USGS/WSC/CDEC source page)
- **Flow** shows discharge + unit; if discharge null, shows stage only; if both null, shows `—`
- **Text this** shows `text_key` in monospace — the exact string to text the bot
- **Updated** is relative time from `reading_time`

### Color coding

Row background (subtle, not garish):

| State | Condition | Color |
|---|---|---|
| Low | `discharge < low` | `#fee2e2` (soft red) |
| Good | `low ≤ discharge ≤ high` | `#dcfce7` (soft green) |
| High | `discharge > high` | `#dbeafe` (soft blue) |
| No range | `low` or `high` is null | white / no color |
| Unavailable | discharge null | `#f3f4f6` (grey) |

A one-line legend above the table: `● Low  ● Good  ● High` using the same colors.

### Live refresh

On page load, JS fetches from Supabase REST (`/rest/v1/gauges?select=*&order=name.asc`) using the public anon key. `setInterval` re-fetches every 10 minutes and re-renders the table in place. A small "Updated N min ago" indicator in the page header shows the last fetch time.

No spinner, no loading skeleton — just renders when it has data.

## Aesthetic

- Background: off-white `#f9f7f4`
- Text: near-black `#1a1a1a`
- Accent: none (color lives in the table rows only)
- Font: system stack — `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- Max-width: 900px, centered, generous padding
- Mobile-friendly via simple `width: 100%` table with horizontal scroll if needed
- No animations, no shadows, no rounded corners on the table

## Not in scope

- User accounts / saved gauges
- Historical flow charts
- Runnable judgment ("GOOD / LOW") — flows are shown raw; ranges just color the row
- River descriptions or photos
- Search / filter on the directory page
- SMS / non-InReach channels

## File structure

```
web/
  index.html        explainer page
  gauges.html       gauge directory
  style.css         shared styles
  gauges.js         fetch + render logic for gauges.html
supabase/
  functions/
    refresh-gauges/
      index.ts      Edge Function — polls APIs, upserts gauges table
  migrations/
    001_gauges.sql  CREATE TABLE gauges + RLS (read-only public)
  seed/
    gauges-seed.json  canonical gauge list with low/high/gauge_url/text_key
```

## Success Criteria

- A paddler who hears about the bot can land on lateboof.com and understand it in under 10 seconds
- The gauge directory shows current flows for all supported rivers with correct color coding
- Page loads fast on mobile cell (one Supabase query, no framework)
- Looks homebuilt, not like a product
